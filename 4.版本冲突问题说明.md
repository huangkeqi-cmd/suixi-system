# 随心系统 - 新旧版本冲突问题说明

## 问题现象

从旧版 (`D:\Users\huangkeqi\CodeBuddy\Claw`) 升级到新版后，可能出现以下问题：

1. **旧版随管无法正常查看影像**
2. **新版识别不稳定**
3. **浏览器打开查看器显示空白或错误**

---

## 问题原因分析

### 1. 端口冲突（主要原因）

新旧版本的随管都使用相同的默认端口启动本地服务器：
- `8888`
- `9000`
- `9999`

当新版的随管启动后占用了这些端口，旧版就无法正常启动服务器，导致查看器无法访问。

### 2. 浏览器缓存

浏览器会缓存 viewer 的 HTML 和 JavaScript 文件。如果新旧版本的 viewer 结构不同，浏览器可能加载了错误的缓存版本。

### 3. Windows 符号链接（Junction）残留

随管使用 Windows 的 junction 符号链接来关联外部照片目录而不复制文件。如果链接没有正确清理，可能导致路径混乱。

### 4. PyInstaller 临时文件

如果运行的是打包后的 `.exe` 版本，PyInstaller 会在 `%TEMP%` 目录创建 `_MEIxxxxx` 临时文件夹。多个版本的临时文件可能相互干扰。

### 5. 项目数据格式差异

新旧版本的 `project.json` 数据格式可能存在差异，旧版可能无法正确解析新版生成的数据。

---

## 解决方案

我为你准备了三个工具脚本：

### 1. `诊断随系问题.bat` ⭐ 先用这个

**作用**：诊断当前系统状态，找出具体问题所在

**使用方法**：
1. 双击运行
2. 查看输出结果，根据提示的问题类型选择对应的解决方案

### 2. `清理随系缓存.bat` ⭐ 日常清理用

**作用**：常规清理，释放端口和缓存

**清理内容**：
- 关闭随管相关进程
- 释放端口 8888/9000/9999/8080
- 清理 PyInstaller 临时文件 (`_MEI*`)
- 清理浏览器扩展存储缓存
- 清理旧版 viewer 目录
- 刷新 DNS 缓存

**使用方法**：
1. 关闭所有随管窗口
2. 双击运行此脚本
3. 运行完成后重新打开随管

### 3. `终极修复-强制重置.bat` ⚠️ 深度修复用

**作用**：强力修复，解决顽固问题

**清理内容**：
- 终止所有 Python 和随管进程
- 强制释放所有相关端口
- 深度清理临时文件
- 清理新旧版 viewer 文件夹
- 重置 Winsock 和网络组件
- 清理资源管理器图标缓存
- 清理浏览器代码缓存

**使用方法**：
1. 保存所有工作
2. 双击运行此脚本
3. 按提示重启电脑

## 最佳实践建议

### 日常使用

1. **不要同时运行新旧版本**
   - 使用旧版时，确保新版完全关闭
   - 使用新版时，确保旧版完全关闭

2. **浏览器使用隐私模式测试**
   - Edge/Chrome 按 `Ctrl+Shift+N` 打开隐私窗口
   - 在隐私窗口中访问查看器，避免缓存干扰

3. **定期清理**
   - 每周运行一次 `清理随系缓存.bat`

### 项目迁移

如果需要将旧版项目迁移到新版：

1. 在旧版中导出项目为 `.sxp` 文件
2. 运行清理脚本关闭旧版
3. 在新版中导入 `.sxp` 文件
4. 重新生成 viewer

## 技术细节（供参考）

### 随管工作原理

```
随管 (main.py)
    ↓
生成 viewer 文件夹（包含 index.html 和 project.json）
    ↓
启动 HTTP 服务器（端口 8888/9000/9999）
    ↓
浏览器访问 http://localhost:端口
```

### 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 查看器空白 | 端口被占用 | 运行清理脚本 |
| 图片不显示 | junction 链接失效 | 重新生成 viewer |
| 旧版打不开 | 新版占用了端口 | 关闭新版，运行清理脚本 |
| 识别不稳定 | 浏览器缓存 | 清除缓存或使用隐私模式 |
