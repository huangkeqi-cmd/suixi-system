## 工作流程

### 第一步：现场采集
1. 将 `capture.html` 发送到手机
2. 手机浏览器打开该文件
3. 导入平面图
4. 添加楼层
5. 在平面图上标记拍摄点位
6. 使用相机拍摄照片/录像
7. 记录拍摄时间
8. 导出项目（project.json + floorplan.jpg）

### 第二步：PC端处理
1. 运行 `python processor.py`
2. 点击"导入项目"，选择 project.json 和平面图
3. 点击"导入照片"，选择影像文件夹
4. 系统自动根据时间戳匹配
5. 手动调整未匹配的点位
6. 点击"生成查看器"

### 第三步：成果查看
1. 点击"启动服务"
2. 扫描二维码在手机端查看
3. 或直接在浏览器打开

## 安装运行

### 环境要求
- Python 3.7+
- 现代浏览器（Chrome/Safari/Edge）

### 安装依赖
```bash
pip install -r requirements.txt
```

### 运行处理工具
```bash
python processor.py
```

## 文件说明

| 文件 | 说明 |
|------|------|
| `capture.html` | 采集端网页，可直接在手机浏览器打开 |
| `processor.py` | PC端处理工具 |
| `requirements.txt` | Python依赖 |
| `project.json` | 采集端导出的项目数据 |
| `floorplan.jpg` | 平面图图片 |

## 数据格式

### project.json
```json
{
  "schemaVersion": "3.0",
  "projectName": "项目名称",
  "floors": [
    {
      "id": "楼层ID",
      "name": "1F",
      "markers": [
        {
          "id": "点位ID",
          "x": 0.5,
          "y": 0.5,
          "status": "captured",
          "captureTime": "2024-01-22T14:30:00"
        }
      ]
    }
  ]
}
```

## 使用技巧

1. **拍摄时校准时间**：如果相机和手机时间不一致，使用校准功能
2. **文件命名**：建议相机设置为包含时间戳的文件名格式
3. **楼层规划**：提前规划好楼层，按顺序拍摄
4. **点位编号**：采集端自动按顺序编号，保持拍摄顺序与编号一致

## 注意事项

1. 采集端使用 localStorage 存储数据，清除浏览器数据会丢失
2. 建议定期导出项目备份
3. 导出的 project.json 和 floorplan.jpg 需放在同一文件夹
4. 生成的 viewer 文件夹可独立部署到任意 Web 服务器
